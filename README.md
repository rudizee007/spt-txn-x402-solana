# SPT-Txn × x402 on Solana

**The authorization layer for the agent economy.**

x402 answers *"did the money move?"* It says nothing about *"was this agent
**allowed** to move it — on whose behalf, within what limit, under what policy,
and can that authority be delegated and revoked?"* That gap is where a
compromised or prompt-injected agent does real damage.

**SPT-Txn fills that gap.** It is a scoped, provable, transaction-bound
authorization token. Authority exists only inside a short-lived token bound to
**one declared payment**, on **one resource**, for **one accountable human**,
verified **offline in well under a millisecond with no call home**. A compromised
agent holds a token that is cryptographically useless for any payment other than
the one it declared.

This repository is the **Solana + x402 integration** of that engine, built on the
open [SPT-Txn reference implementation](https://github.com/rudizee007/spt-txn-poc).

---

## The one-line pitch

> **x402 moves the money. SPT-Txn proves the agent was allowed to — offline,
> per-transaction, with a human anchor and a tamper-evident receipt.**

---

## What it does

An AI agent hits a resource, gets an **HTTP 402 Payment Required**, and *before*
it signs anything on Solana:

1. The **SPT-Txn gate** mints a `CAT → CT → SPT-Txn` chain bound to that exact
   payment (amount, pay-to, resource) and the agent's capability ceiling.
2. If the payment is **within scope**, the gate **ALLOWS** it and the settler
   submits the USDC/SOL transfer on Solana, writing an immutable, **zero-knowledge
   human anchor on-chain via the SPL Memo program** — no PII on the ledger.
3. If the payment is **over scope, expired, or the delegation chain widens**, the
   mint is refused: the gate **DENIES** and the agent must not sign.

Every decision emits a **signed receipt** as a byproduct — compliance evidence is
generated by enforcement, not by an after-the-fact audit.

## Repository layout

```
gate/       Option 1 — off-chain SPT-Txn x402 authorization gate (Go).
            Decides IF the agent may pay, before any signature. Imports the
            open SPT-Txn engine; no Solana SDK in this path.
settle/     Solana settlement client (isolated Go module). Submits the transfer
            and writes the human anchor on-chain via SPL Memo. Devnet by default.
onchain/    Option 2 — Anchor (Rust) program that enforces the SPT-Txn proof
            on-chain (near-path target).
demo/       3-minute demo storyboard + scripts for the submission video.
docs/       Architecture, build plan, and monetization one-pager.
```

## Design invariants (non-negotiable)

- **Deny by default, fail closed.** Timeout, malformed token, unreachable
  revocation, over-scope → **DENY** with an evidence record explaining why.
- **No ambient authority.** Authority lives only in the short-lived,
  transaction-scoped token — never in a role, a network location, or a
  long-lived key.
- **Attenuation is monotonic and offline-verifiable.** Every delegation step can
  only *narrow* authority. Child TTL strictly less than parent.
- **No custom cryptography.** Standard, audited primitives only
  (`crypto/ed25519`, Solana's Ed25519 precompile, SPL Memo). No hand-rolled
  constructions.
- **No PII on-chain.** The human anchor is a zero-knowledge commitment, not an
  identity.

## Security note

Devnet keypairs and any `*.json` key material are **git-ignored** and must never
be committed. The settler reads keys from the environment only
(`SOL_OPERATOR_KEY`), never from a flag or the repo. Mainnet requires an explicit
confirmation.
